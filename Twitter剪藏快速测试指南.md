# 🐦 Twitter剪藏快速测试指南

## 🎯 测试目标
解决Twitter/X页面剪藏时只保存图片，不保存文字内容的问题。

## 🚀 快速步骤

### 步骤1: 打开页面
1. 打开目标Twitter页面:
   ```
   https://x.com/treydtw/status/2012825579934962004
   ```

### 步骤2: 等待加载
- 等待页面完全加载（所有内容显示）
- 如果需要，向下滚动确保内容完全加载

### 步骤3: 使用测试脚本
1. 按F12打开开发者工具
2. 切换到"Console"标签页
3. 复制并粘贴 `Twitter页面测试脚本.js` 中的代码
4. 按Enter运行
5. 查看控制台输出

### 步骤4: 查看结果
寻找以下输出：
- ✅ 选择器找到元素
- 📝 文本内容预览
- 🎯 发现有效内容

### 步骤5: 使用Clipper
1. 点击浏览器扩展的Clipper图标
2. 选择"完整文章"模式
3. 点击"保存"

### 步骤6: 检查结果
1. 在Obsidian中打开生成的文件
2. 检查是否包含文字内容
3. 如果没有，继续下面的调试步骤

## 🔧 调试步骤

### 如果剪藏失败：

#### 方案A: 使用浏览器脚本测试
```javascript
// 在控制台运行，快速检查
const tweet = document.querySelector('[data-testid="tweet"]');
if (tweet) {
    console.log('✅ 找到tweet元素');
    console.log('内容:', tweet.textContent.substring(0, 200));
} else {
    console.log('❌ 未找到tweet元素');
}
```

#### 方案B: 更新配置
已经更新了 `data.json` 文件，包含：
- Twitter专用选择器: `[data-testid='tweet']`, `[data-testid='tweetText']`, `div[role='article']`
- 动态内容支持: `waitForContent: 3000`
- 滚动到内容: `scrollToContent: true`

#### 方案C: 手动提取
如果自动提取失败，可以：
1. 手动复制文字内容
2. 手动保存图片
3. 在Obsidian中手动创建文件

## 🚨 故障排除

### 常见问题：

| 问题 | 解决方案 |
|------|----------|
| 只有图片没有文字 | 运行测试脚本检查选择器 |
| 内容不完整 | 增加等待时间到5000ms |
| 提取失败 | 检查网络连接和权限 |
| 格式混乱 | 检查CSS类名是否变化 |
