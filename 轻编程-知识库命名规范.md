## 📝 文件命名规范

### 命名格式

**基础格式**:
```
YYYY-MM-DD-[主题]-[标题]
```

**示例**:
```
2026-01-19-vibe-coding-什么是vibe-coding.md
2026-01-19-claude教程-claude-cli基础使用.md
2026-01-19-实战项目-我的第一个vibe-coding项目.md
```

### 特殊前缀

**状态前缀**:
- `[WIP]` - 进行中
- `[DONE]` - 已完成
- `[IDEA]` - 想法
- `[Q&A]` - 问题解答
- `[TEMPLATE]` - 模板

**示例**:
```
[WIP]2026-01-19-claude教程-高级功能.md
[IDEA]2026-01-19-创意项目-vibe-coding小游戏.md
```

## 🗂️ 内容分类标准

### 按学习阶段

**入门级**:
- 标签：`#入门`
- 特点：概念解释、基础操作
- 受众：完全新手

**进阶级**:
- 标签：`#进阶`
- 特点：深入原理、复杂应用
- 受众：有基础的学习者

**高级级**:
- 标签：`#高级`
- 特点：专业技巧、优化提升
- 受众：有一定经验者

### 按内容类型

**教程类**:
- 标签：`#教程`
- 格式：步骤化、可操作

**概念类**:
- 标签：`#概念`
- 格式：解释性、理论性

**实战类**:
- 标签：`#实战`
- 格式：项目化、案例化

**问答类**:
- 标签：`#问答`
- 格式：问题-解答式

## 🎯 质量标准

### 内容质量要求

**准确性**:
- 技术信息准确无误
- 代码示例可运行
- 引用来源可靠

**完整性**:
- 概念解释完整
- 步骤说明清晰
- 示例代码完整

**可读性**:
- 语言简洁明了
- 结构清晰有序
- 格式统一规范

**实用性**:
- 解决实际问题
- 提供可操作方案
- 有具体应用场景

### 更新频率

**定期更新**:
- 每周至少新增2-3篇文章
- 每月整理一次索引
- 每季度优化一次结构

**及时维护**:
- 发现错误立即修正
- 过期内容及时更新
- 新技术及时补充
