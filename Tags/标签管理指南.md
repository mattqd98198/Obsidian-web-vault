# 🏷️ Obsidian标签管理指南

## 标签分类体系

### 📚 内容分类
- #网页 - 所有通过Clipper收集的网页内容
- #剪藏 - 标记为剪藏的文章
- #待阅读 - 需要阅读的内容
- #已读 - 已阅读完成的内容
- #重要 - 高价值内容
- #参考 - 参考资料类内容

### 🎯 技术分类
- #前端 - 前端开发相关
- #后端 - 后端开发相关
- #全栈 - 全栈开发
- #移动端 - 移动应用开发
- #AI - 人工智能相关
- #数据 - 数据科学相关
- #安全 - 网络安全相关
- #运维 - 运维相关

### 💼 工具分类
- #工具 - 各种开发工具
- #框架 - 开发框架
- #库 - 开发库
- #IDE - 集成开发环境
- #版本控制 - Git等版本控制工具
- #部署 - 部署相关

### 🎨 设计分类
- #设计 - 设计相关
- #UI - 用户界面设计
- #UX - 用户体验设计
- #原型 - 原型设计
- #视觉 - 视觉设计

### 📖 学习分类
- #教程 - 教程类内容
- #文档 - 官方文档
- #书籍 - 电子书籍
- #课程 - 在线课程
- #认证 - 认证考试

### 🚀 项目分类
- #个人项目 - 个人项目相关
- #工作项目 - 工作项目相关
- #开源项目 - 开源项目相关
- #创业项目 - 创业项目相关

### 🔍 优先级分类
- #高优先级 - 需要立即处理
- #中优先级 - 需要近期处理

- #总结 - 经验总结

## 标签使用规则

### 基础规则
1. 每个笔记至少要有2-3个标签
2. 标签要有层次关系，从大类到小类
3. 标签命名要简洁明了
4. 避免使用过于具体的标签

### 组合示例
```
#前端 #Vue.js #教程 #已读 #高价值

#AI #机器学习 #深度学习 #论文 #待阅读

#工具 #VS Code #效率 #个人项目
```

### 自动标签配置
Clipper配置文件中已预设了自动标签功能：

```json
{
  "autoTags": {
    "github.com": ["#开发", "#技术", "#编程"],
    "zhihu.com": ["#知识", "#讨论", "#问答"],
    "medium.com": ["#深度", "#英文", "#技术"],
    "sspai.com": ["#教程", "#实用", "#工具"],
    "掘金": ["#技术", "#前端", "#开发"],
    "知乎": ["#知识", "#讨论", "#观点"]
  }
}
```

## 标签搜索技巧

### 基础搜索
```markdown
# 搜索所有技术相关内容
#技术

# 搜索特定技术栈
#前端

# 搜索多个标签
#前端 #Vue.js
```

### 组合搜索
```markdown
# 搜索已读的技术文章
#技术 #已读

# 搜索待阅读的AI内容
#AI #待阅读

# 搜索高优先级的前端教程
#前端 #教程 #高优先级
```

### Dataview查询
```dataview
TABLE title, created, tags
FROM "01-Inbox/网页剪藏"
WHERE contains(tags, "#技术")
  AND contains(tags, "#待阅读")
SORT created DESC
```

## 标签维护建议

### 定期清理
1. 每月检查一次标签使用情况
2. 删除无用的重复标签
3. 合并相似的标签
4. 更新标签分类体系

### 标签统计
```dataview
TABLE
  count(rows) as 数量,
  join(rows.tags, ", ") as 标签列表
FLATTEN tags as tag
GROUP BY tag
SORT 数量 DESC
```

### 新标签添加流程
1. 确定标签的大类归属
2. 检查是否有相似标签
3. 确认标签命名规范
4. 更新Clipper自动标签配置
5. 在相关笔记中添加标签
